# Test Plan – Login Functionality

## Application Under Test
- **Name:** Practice Test Automation – Login  
- **URL:** https://practicetestautomation.com/practice-test-login/  

---

## Objective
The purpose of this test plan is to define the approach for testing the login functionality and to ensure correct behavior for both valid and invalid login attempts.

---

## Scope
- Login form validation  
- Authentication with valid credentials  
- Handling of invalid credentials  
- Validation messages for incorrect or missing input  
- Error message display after failed login attempts  
- User redirection after successful login  

---

## Test Environment
- **Browser:** Chrome  
- **Operating System:** macOS

## Entry and Exit Criteria

### Entry Criteria
- Login page is accessible  
- Test cases are prepared  

### Exit Criteria
- All planned test cases are executed  
- Test results are reviewed  
- Identified defects are documented

## Test Deliverables
- Manual test cases  
- Bug reports
